# 割雪 | Gexue — Hugo 个人网站

## 项目概况

- 独立游戏开发者个人网站，域名 `gexue.dev`
- **技术栈**：Hugo 0.163.3 (extended) + PaperMod 主题 (git submodule)
- **语言**：zh-cn（中文），强制深色模式
- **部署**：GitHub Actions → GitHub Pages，push `main` 分支自动触发
- **本地推送**：运行 `gitpush.bat`（`git add .` → commit → push）
- **仓库地址**：`E:\Project\Gexue`

## 目录结构

```
E:\Project\Gexue\
├── .github\workflows\hugo.yml    # CI：peaceiris/actions-hugo@v3 构建
├── archetypes\default.md         # Hugo 原型（TOML frontmatter）
├── content\devlog\               # 开发日志（主内容区，mainSections）
│   ├── _index.md                 # 列表页标题
│   └── 2026-06-27.md             # 第一篇文章「起点」
├── themes\PaperMod\              # 主题子模块，不要直接修改
├── hugo.toml                     # 站点配置
├── gitpush.bat                   # 手动推送脚本
├── CNAME                         # 自定义域名
└── .gitmodules
```

## hugo.toml 关键配置

- `baseURL`: `https://gexue.dev/`
- `theme`: `PaperMod`, `paginate`: 20
- `DateFormat`: `"2006-01-02"` → 纯数字日期（如 `2026-06-27`），替代默认的 "June 27, 2026"
- `showReadingTime`: `false` — **已关闭阅读时间**。原因：Hugo `.ReadingTime` 基于 `.WordCount`（按空格分词），中文无空格导致无论文章多长都显示 `1 min`。PaperMod 没有 relative time（"X 天前"）功能，那个 `1 min` 是阅读时间而非发布时间距离
- 菜单：`开发日志` (/devlog/) + `bilibili` (外链)
- 日期排序：`date`, `publishDate`, `lastmod`

## 文章格式

- 路径：`content/devlog/YYYY-MM-DD.md`
- Frontmatter（TOML）：`+++ title, date, draft +++`
- 原型用 `archetypes/default.md`

## 已知约束

- **不要修改 `themes/PaperMod/`** 内的文件（git submodule）
- **不要修改 `.github/workflows/hugo.yml`** 除非明确要求
- 需要自定义布局时，在项目 `layouts/` 目录下覆盖对应文件
- 构建前确保没有残留 `hugo server` 进程在运行（会干扰构建缓存）
- Hugo 缓存目录：`$HOME/AppData/Local/hugo_cache`，`--gc` 需要该目录可写

## 常见问题排查

- **"1 min" 问题**：非发布时间，是 PaperMod 的阅读时间标记，因中文分词失效。已通过 `showReadingTime = false` 关闭
- **构建不生效**：检查 `hugo server` 是否在后台运行（`Get-Process hugo`），有则 `Stop-Process` 杀掉后删除 `public/` 重建
- **文件写入**：`apply_patch` 可能静默失败，用 PowerShell `[System.IO.File]::WriteAllText` + UTF-8 兜底
- **TOML 编码**：PowerShell heredoc 可能导致中文乱码，用 `WriteAllText` + `UTF8Encoding($false)`
